# Rule: Review Memory

## Quy tắc bắt buộc

- Mỗi lần hoàn thành một prompt, một cửa sổ làm việc, một plan, một phase, hoặc một task, phải cập nhật file review để lưu “bộ nhớ vận hành”.
- Nội dung review phải ngắn gọn, đủ để lần sau đọc lại biết đã làm gì và cần chú ý gì.
- Không ghi bí mật, token, credential, hoặc dữ liệu nhạy cảm vào review.

## File chuẩn

- `.claude/.claude/review/REVIEW.md`

## Nội dung bắt buộc trong mỗi lần cập nhật

- Việc đã làm.
- File đã thay đổi.
- Kết quả kiểm tra/verification.
- Rủi ro còn lại.
- Bài học rút ra.
- Bước tiếp theo đề xuất.
